Reports from the course Software Development for the Web

S01

Did you before know about the techniques Git, GitHub, Markdown and/or GitHub Pages?
I had a limited knowledge before this introduction. We have used github before in previous courses as our mandatory version control. I have not used Git in the sense of console but rather relied on the GitHub Desktop App before this. I had never heard of Github Pages before this. I have used markdown before, as Discord, for example supports it.

Have you ever created websites before?
We have used html for web services, but that was a single lab. I have never created proper a webpage before.

Briefly explain your experience and knowledge of web application development.
My knowledge of web development is very limited. I have had brief experience with web services in the past. I have done a little bit of Javascript. I would say that my knowledge is not within web application development, but just network technology briefly touching on the subject.

What is your TIL for this course section? Today I learned about GitHub Pages. I never knew this existed, despite using GitHub for a long time. I had always wanted to try and make a dashboard for my Discord bot that I made to compete with the successful and in my opinion terribly coded bot Dynobot. I find that, while not a permanent solution, that GitHub pages would be an excellent to test this out. I also further developed my knowledge with Git console commands. I did not know about tags before this.

S02

Have you any previous experience of HTML, CSS and/or JavaScript?
We've had previous courses which have touched on HTML, but only very briefly. CSS has been part of many things, however, I've avoided it as it's been largely optional for most projects we have done. We did not, however, receive any real teaching on it either. I have used Javascript previously, as I developed a Discord bot for a community, using Node and their API. I have read documentation beforehand on it, as I had an interest to learn.

Explain the role of HTML, CSS and JavaScript in web development.
HTML provides the requested resource, and presents it to the user. CSS controls the visual aspect of the presentation. Javascript makes the website dynamic, influencing the elements provided by the HTML. Together, all of these present what we know as a modern website, on a basic level at any rate.

Give a brief explanation of how the browser, the HTTP protocol and the web server interacts.
Browser requests the documents through the HTTP Protocol, from the web server. It downloads and represents these to the user. This also includes any resources required.

What is your TIL for this course section? I'm unsure if I learnt much in this part, as we're mostly starting out so far. Perhaps I learnt a bit about CSS, but most of these concepts are familiar to me, even if the implementation may be new. I have had an interest in Javascript, so learning a bit more of the basics is good. I also learnt that programming languages may be widely different, but working with them gives a sense of familiarity

S03

Do you have any previous experience of client side JavaScript?
No, I do not have any experience with client side JS. However, with the little JS experience I have it was fairly easy to pick up. Most of the fundamental knowledge of programming still applies here, making it easier to understand.

Can you compare and relate the JavaScript language to any other language you know?
It's hard to compare languages, especially when we have not worked with many in the past. I would like to say that it is similar in terms of writing to Java, more than I expected. I feel like it's easier to pick Javascript up, even though it's fundamentally different, to for example C# which is supposedly more similar to Java. I think the hardest part to grasp for me was using JSON, as it has always confused me when I started out with JS. The strangest part would be the variable declaration, as it is very different. Working with arrays are extremely similar, maybe even easier in JS due to these things.

Describe how you worked with the coding exercise, what grade do you aim for and how did your code turn out to be?
I approached with a minimal-effort mentality, as I did not have the energy to put the effort in. I was told that the online API would not work, and as such I did not spend time trying it, but rather using the JSON file. I aimed for grade 3, as I procrastinated and waited until the last moment to finish all of the Assignments. My code is basic, but it does the job.

What is your TIL for this course section? As the main part was the fetch(), I feel like I learnt a lot about getting and reading JSON responses. It also gave me an idea on some of the projects I've had lined up, as they could benefit from this too. I also gained more knowledge of CSS, and how to manipulate it through Javascript.
